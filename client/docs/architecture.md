# Архитектурные принципы

## 1. Философия

Главное: **low coupling + high cohesion**.
Модули независимы, решают одну задачу → проще тестировать, переиспользовать и менять.

## 2. Стек

React + TypeScript + Vite
React Router
Tailwind CSS
Zustand
ESLint + Prettier

## 3. Архитектура

Используется **Feature-Sliced Design (FSD)** — строгая декомпозиция по слоям ответственности.

## 4. Структура

```
src/
app/      — инициализация
pages/    — страницы
widgets/  — крупные UI-блоки
features/ — бизнес-логика
entities/ — сущности
shared/   — переиспользуемое (без бизнес-логики)
```

**Правило**: импорт только вниз по слоям.

## 5. Состояние

- Глобальное → Zustand
- Локальное → useState / useReducer
- Персистентное → Zustand persist

❌ React Context для бизнес-состояния запрещён.

## 6. API

- Всё API → `shared/api`
- Никаких `fetch` вне этого слоя
- Моки → `shared/api/mocks`

## 7. Ключевые архитектурные правила

- Изоляция фич: Импорты между разными фичами (`feature → feature`) **строго запрещены**.
- shared — без бизнес-логики и зависимостей вверх
- Каждый модуль имеет публичный API (`index.ts`)
